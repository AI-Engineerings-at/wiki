# Skills — AI Engineering Toolkit

> 49 Eigenbau-Skills | 6 Kategorien | 2 Repos | Quality Gate fuer jeden Skill

Alle Skills sind **EIGENBAU** — entwickelt und maintained von AI Engineering.
Sie laufen als Claude Code Auto-Load Skills in `.claude/skills/` Verzeichnissen.

## Was sind Skills?

Skills sind wiederverwendbare Prompt-Templates fuer Claude Code. Jeder Skill besteht aus einer `SKILL.md` Datei mit YAML Frontmatter (Metadata) und einem Body (Anweisungen). Claude Code laedt sie automatisch wenn du im entsprechenden Verzeichnis arbeitest.

**Offizieller Anthropic Standard + Custom Extensions:**
- Anthropic Felder: `name`, `description`, `instructions`, `command_injection`
- Unsere Erweiterungen: `version`, `requires`, `produces`, `error-refs`, `learning-refs`
- String Substitutions: `$ARGUMENTS`, `${CLAUDE_SKILL_DIR}`, `${workspaceFolder}`

## Quality Gate

Jeder Skill muss diese Anforderungen erfuellen:

- [ ] `SKILL.md` mit vollstaendigem YAML Frontmatter
- [ ] `name` und `description` gesetzt
- [ ] Mindestens 1 Trigger-Keyword definiert
- [ ] Body mit klaren Anweisungen (kein Filler)
- [ ] Getestet im jeweiligen Repo-Kontext
- [ ] Dokumentiert im Skill-Katalog

## Skill-Lifecycle

```
CREATE → Idee + /skill-creator → SKILL.md scaffolden
  ↓
DEVELOP → Body schreiben, Frontmatter setzen
  ↓
TEST → Im Repo ausfuehren, Ergebnis pruefen
  ↓
CATALOG → In Skill-Katalog eintragen
  ↓
MAINTAIN → Version bumpen, Error/Learning-Refs pflegen
  ↓
ARCHIVE → Deprecated-Flag setzen, Ersatz dokumentieren
```

## Skill-Architektur in Detail

### Frontmatter Fields (Vollreferenz)

| Feld | Typ | Pflicht | Beschreibung |
|------|-----|---------|-------------|
| `name` | String | Ja | Eindeutige ID (kebab-case) |
| `description` | String | Ja | Kurzbeschreibung mit Trigger-Keywords |
| `version` | String | Ja | Semantic Versioning (X.Y.Z) |
| `requires` | Array | Ja | Dependencies (Rules, Skills, Tools) |
| `produces` | Array | Ja | Output-Typen (Datei, Notification, etc.) |
| `disable-model-invocation` | Boolean | Nein | true = nur manuell invokierbar |
| `user-invocable` | Boolean | Nein | false = nur von Claude invokierbar |
| `model` | String | Nein | haiku\|sonnet\|opus |
| `context` | String | Nein | fork = isolierter Subagent-Context |
| `agent` | String | Nein | Explore\|Plan\|general-purpose |
| `allowed-tools` | Array | Nein | Whitelist von Tools |
| `error-refs` | Array | Nein | Referenzen zu ERROR_REGISTRY.md |
| `learning-refs` | Array | Nein | Referenzen zu LEARNINGS_REGISTRY.md |
| `last-verified` | String | Ja | Datum der letzten Verification (YYYY-MM-DD) |

### String Substitutions (dynamische Werte)

Claude Code ersetzt diese automatisch zur Laufzeit:

| Variable | Beschreibung | Beispiel |
|----------|-------------|---------|
| `$ARGUMENTS` | Alle Benutzereingaben nach Skill-Name | `/skill-name arg1 arg2` → `arg1 arg2` |
| `$0` | Erstes Argument | `arg1` |
| `$1` | Zweites Argument | `arg2` |
| `$N` | N-tes Argument | `$3` für drittes Argument |
| `${CLAUDE_SESSION_ID}` | Unique Session Identifier | `abc-def-ghi-jkl` |
| `${CLAUDE_SKILL_DIR}` | Absoluter Pfad zum Skill-Verzeichnis | `/Users/name/.claude/skills/skill-name/` |
| `${CLAUDE_SESSION_ID}` | Session eindeutige ID | für Logging/Correlation |

### Dynamic Context Injection

Shell-Commands werden VOR dem Skill ausgeführt mit der `` !`command`! `` Syntax:

```markdown
---
command_injection: "git log --oneline -5"
---

## Aktueller Status
!`curl -s http://api.example.com/status | jq .`!

Dieser Wert wird eingefügt bevor Claude den Body sieht.
```

Nutzbar für:
- Aktuelle Systemzustände
- Git History
- Deployment Status
- API-Daten

## Skill-Best Practices

### Ein Skill, ein Job

Erstelle fokussierte Skills:
- ✅ `/deploy` — Deploy VG
- ✅ `/test-suite` — Fuehre Tests aus
- ❌ `/do-everything` — macht alles (zu viel)

### Trigger-Keywords Explicit

Description sollte klare Keywords enthalten:

```yaml
description: "Deploy Voice Gateway oder Dashboard — Trigger: deploy, VG, dashboard"
```

Claude nutzt die Keywords um den Skill auto-zu-laden.

### Error-Handling dokumentieren

Schreib die Top-5 Fehler und Fixes:

```markdown
## Fehler-Handling

- SSH Timeout → Retry nach 10s, max 3x
- Build Failure → Logs zeigen, nicht wieder bauen
- Health Check failed → Manual Pruefen noetig
```

### Keine Secrets im Skill

```markdown
# RICHTIG
Lade Credentials ueber: vault.py get <service> <key>

# FALSCH
token="sk-abc123def456"  ❌ NIEMALS
```

## Skill-Discovery & Auto-Loading

Claude Code:
1. Laedt **alle Skill-Descriptions** in Context (so Claude weiss was verfuegbar ist)
2. Laedt **nur den Body** eines Skills wenn:
   - Benutzer `/skill-name` ausfuehrt, ODER
   - Claude das Skill auto-laedt basierend auf Description

**Beispiel:**
```
Benutzer: "How does the auth system work?"
Claude sieht: description "explain-code: explains code with diagrams"
→ Auto-laedt den Skill
→ Gibt Diagram + Explanation
```

## Skills in Plugins verteilen

Skills koennen als Teil von Plugins distribuiert werden:

```
my-plugin/
├── .claude-plugin/plugin.json
└── skills/
    └── code-review/
        └── SKILL.md
```

Benutzer installieren:
```bash
/plugin install my-plugin
/code-review-plugin:code-review  # Mit Namespace
```
