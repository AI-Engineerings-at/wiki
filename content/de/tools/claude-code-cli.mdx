# Claude Code CLI — Vollstaendige Referenz

> Dieses Dokument ist eine REFERENCE — nicht ein Guide, nicht ein Ueberblick. Jedes Flag, jedes Kommando, jede Option wird dokumentiert. Basiert auf der offiziellen Anthropic Dokumentation (Stand: März 2026).

## Installation

### Unterstuetzte Plattformen

Claude Code ist verfuegbar fuer:
- **macOS** 13.0 oder neuer (Intel und Apple Silicon)
- **Linux** x86_64 und aarch64 (glibc-Systeme)
- **Alpine Linux** und musl-basierte Distributionen (spezielle Behandlung noetig)
- **Windows** 10/11 mit Git Bash
- **WSL1/WSL2** auf Windows

### Installationsmethoden

#### Shell-Installer (macOS / Linux)

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Das Skript:
- Prueft Systemanforderungen
- Erkennt CPU-Architektur automatisch
- Installiert nach `~/.local/bin/claude`
- Erkennt Ihr Shell-Profil und ergaenzt es mit PATH

Nach Installation:
- Terminal neu starten ODER
- `source ~/.zshrc` / `source ~/.bashrc` ausfuehren

#### PowerShell-Installer (Windows)

```powershell
irm https://claude.ai/install.ps1 | iex
```

Das Skript:
- Prueft Systemanforderungen
- Erkennt Windows-Architektur
- Installiert nach `%USERPROFILE%\.local\bin\claude.exe`
- Ergaenzt User PATH Umgebungsvariable

Nach Installation: Terminal neu starten.

#### CMD-Installer (Windows)

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

#### Homebrew (macOS)

```bash
brew install --cask claude-code
```

#### WinGet (Windows)

```powershell
winget install Anthropic.ClaudeCode
```

#### NPM (alle Plattformen, deprecated)

```bash
npm install -g @anthropic-ai/claude-code
```

> NPM-Installation ist deprecated. Verwenden Sie den Shell/PowerShell-Installer statt dessen. Sie sind 3-4x schneller.


## CLI Kommandos

| Kommando | Beschreibung | Beispiel |
|----------|-------------|---------|
| `claude` | Starte interaktive Session | `claude` |
| `claude "query"` | Interaktive Session mit initialem Prompt | `claude "erklaere dieses Projekt"` |
| `claude -p "query"` | Print Mode: Query via SDK, dann Exit | `claude -p "erklaere diese Funktion"` |
| `cat file \| claude -p "query"` | Print Mode mit gepiptem Content | `cat logs.txt \| claude -p "analysiere"` |
| `claude -c` | Continue: Letzt Session im aktuellen Verzeichnis fortsetzen | `claude -c` |
| `claude -c -p "query"` | Continue via SDK | `claude -c -p "pruefen auf Fehler"` |
| `claude -r "<session>" "query"` | Resume: Session nach ID oder Name fortsetzen | `claude -r "auth-refactor" "Finish PR"` |
| `claude update` | Update auf neueste Version | `claude update` |
| `claude auth login` | Anmelden bei Anthropic Account | `claude auth login --console` |
| `claude auth logout` | Abmelden | `claude auth logout` |
| `claude auth status` | Authentifizierungsstatus (JSON oder Text) | `claude auth status --text` |
| `claude agents` | Alle konfigurierten Subagents auflisten | `claude agents` |
| `claude mcp` | Model Context Protocol Server konfigurieren | `claude mcp add server` |
| `claude remote-control` | Remote Control Server starten (Server Mode) | `claude remote-control --name "Projekt"` |

## CLI Flags — Komplette Referenz

### Allgemeine Flags

| Flag | Beschreibung | Standard | Beispiel |
|------|-------------|----------|---------|
| `--version`, `-v` | Versionsnummer ausgeben | - | `claude -v` |
| `--verbose` | Verbose Logging aktivieren (vollstaendige Turn-by-Turn Ausgabe) | false | `claude --verbose` |
| `--help`, `-h` | Hilfe anzeigen | - | `claude --help` |
| `--debug` | Debug Mode mit optionaler Kategorie-Filterung | false | `claude --debug "api,mcp"` |

Debug-Kategorien:
- `api` — API-Aufrufe
- `mcp` — MCP Server
- `hooks` — Lifecycle Hooks
- `file` — Datei-Operationen
- `statsig` — Feature Flags

Negation mit `!` (z.B. `"!statsig,!file"`).

### Session Management

| Flag | Beschreibung | Standard | Beispiel |
|------|-------------|----------|---------|
| `--name`, `-n` | Session einen Display-Namen geben (in `/resume` und Terminal-Titel) | auto | `claude -n "mein-feature"` |
| `--continue`, `-c` | Letzt Session im aktuellen Verzeichnis laden | false | `claude -c` |
| `--resume`, `-r` | Session nach ID oder Name fortsetzen, oder picker anzeigen | - | `claude -r auth-refactor` |
| `--fork-session` | Neue Session-ID erstellen statt alte zu reuses (mit `--resume`) | false | `claude -r abc123 --fork-session` |
| `--session-id` | Spezifische Session-ID verwenden (muss valid UUID sein) | auto | `claude --session-id "550e8400..."` |
| `--no-session-persistence` | Sessions nicht auf Disk speichern (Print Mode nur) | false | `claude -p --no-session-persistence "query"` |
| `--from-pr` | Sessions von spezifischem GitHub PR fortsetzen (PR# oder URL) | - | `claude --from-pr 123` |
| `--remote` | Neue Web-Session auf claude.ai mit Task-Description | false | `claude --remote "Fix Login Bug"` |
| `--remote-control`, `--rc` | Remote Control aktivieren (control von claude.ai) | false | `claude --remote-control "Projekt"` |
| `--teleport` | Web-Session ins lokales Terminal zurueckbringen | false | `claude --teleport` |

### Model und Agent Configuration

| Flag | Beschreibung | Standard | Beispiel |
|------|-------------|----------|---------|
| `--model` | Model fuer diese Session setzen (haiku, sonnet, opus oder Full Name) | Parent Session | `claude --model claude-sonnet-4-6` |
| `--effort` | Effort-Level setzen (low, medium, high, max) nur Opus 4.6 | medium | `claude --effort high` |
| `--fallback-model` | Fallback Model wenn Default ueberlastet (Print Mode nur) | - | `claude -p --fallback-model sonnet "query"` |
| `--agent` | Custom Agent fuer diese Session | default | `claude --agent my-custom-agent` |
| `--agents` | Custom Subagents dynamisch via JSON definieren | - | `claude --agents '{"reviewer":{...}}'` |

### Permission und Security

| Flag | Beschreibung | Standard | Beispiel |
|------|-------------|----------|---------|
| `--permission-mode` | Begin in spezifischem Permission Mode (default, plan, ask) | default | `claude --permission-mode plan` |
| `--dangerously-skip-permissions` | Permission Prompts ueberspringen (VORSICHT!) | false | `claude --dangerously-skip-permissions` |
| `--allow-dangerously-skip-permissions` | Permission-Bypassing aktivieren (mit `--permission-mode`) | false | `claude --permission-mode plan --allow-dangerously-skip-permissions` |
| `--permission-prompt-tool` | MCP Tool fuer Permission Prompts im Non-Interactive Mode | - | `claude -p --permission-prompt-tool mcp_auth_tool "query"` |

### Tool Control

| Flag | Beschreibung | Standard | Beispiel |
|------|-------------|----------|---------|
| `--tools` | Verfuegbare Built-In Tools einschraenken | alle | `claude --tools "Bash,Edit,Read"` |
| `--disallowedTools` | Tools entfernen (als Blacklist) | - | `claude --disallowedTools "Bash(git log *)"` |
| `--allowedTools` | Tools die ohne Permission laufen | - | `claude --allowedTools "Bash(git log *)" "Read"` |

Syntax:
- `"Bash"` — alle Bash Commands
- `"Bash(git log *)"` — nur `git log` Pattern-Matching
- `"Read"` — Read Tool vollstaendig
- `""` — alle Tools deaktivieren

## Exit Codes

| Code | Bedeutung | Beispiel |
|------|-----------|---------|
| `0` | Erfolgreich | Command completed |
| `1` | Allgemeiner Fehler | Authentifizierung fehlgeschlagen, Invalid Input |
| `2` | Command Line Parse Error | Invalid Flags, Syntax Error |
| `42` | Context Limit Exceeded | Token Limit ueberschritten (Print Mode) |
| `99` | Interner Fehler | Crash, Unerwartete Exception |

Beispiele:
```bash
claude auth status
echo $?  # Gibt 0 wenn logged in, 1 wenn nicht
```

## Haeufige Fehlerbehandlung

### Installation

| Error | Loesung |
|-------|---------|
| `command not found: claude` | PATH nicht updated — siehe "Verify your PATH" in Troubleshooting |
| `syntax error near unexpected token '<'` | Install Script returned HTML (nicht Script) — versuchen Sie Homebrew oder WinGet |
| `curl: (56) Failure writing output to destination` | Verbindung abgebrochen — retry oder alternative Methode |
| `Killed` auf Linux (low-memory) | Swap Space hinzufuegen: `sudo fallocate -l 2G /swapfile && sudo swapon /swapfile` |
| `TLS connect error` oder `SSL/TLS secure channel` | CA Certificates updaten oder Proxy konfigurieren |
| `Failed to fetch version` | Network blockiert google cloud storage — Proxy/VPN versuchen |

### Authentication

| Error | Loesung |
|-------|---------|
| `OAuth error: Invalid code` | Login Code ist abgelaufen — retry schnell |
| `403 Forbidden` | Subscription inactive oder API Key invalid — `/status` pruefen |
| "This organization has been disabled" | `ANTHROPIC_API_KEY` Env-Var ueberschreibt Subscription — `unset ANTHROPIC_API_KEY` |
| `Not logged in` | OAuth Token abgelaufen — `/login` ausfuehren |

### Performance

| Error | Loesung |
|-------|---------|
| High CPU/Memory usage | `/compact` verwenden, neustart zwischen Tasks, `.gitignore` aktualisieren |
| Command hangs/freezes | `Ctrl+C` druecken zum abbrechen, Terminal neu starten |
| Search/Discovery broken | System `ripgrep` installieren: `brew install ripgrep` |

### Konfiguration

| Error | Loesung |
|-------|---------|
| Malformed settings.json | JSON validieren (VSCode JSON Validator) |
| MCP Server nicht geladen | `claude --debug` ausfuehren, Errors in stderr pruefen |
| Permission Prompts wiederholt | `/permissions` ausfuehren, Rules konfigurieren |

## Best Practices

### Session Management

1. **Named Sessions verwenden** fuer bessere Organization:
   ```bash
   claude -n "feature-auth-refactor"
   ```

2. **Continue statt Resume** fuer gleichen Verzeichnis:
   ```bash
   claude -c  # Statt -r session-id
   ```

3. **Worktrees fuer parallele Work**:
   ```bash
   claude -w feature-1  # Isolierter Branch
   claude -w feature-2  # Anderer Branch
   ```

### Performance

1. **Context regularmaessig compacten**:
   ```bash
   /compact
   ```

2. **Grossere Codebases mit `--add-dir` scoping**:
   ```bash
   claude --add-dir ./src ./tests
   ```

3. **Limit Turns bei API-Kosten-Sensitivitaet**:
   ```bash
   claude -p --max-turns 3 "query"
   ```

### Sicherheit

1. **Niemals Credentials in CLI Flags**:
   ```bash
   # FALSCH
   claude --betas "api_key=secret"

   # RICHTIG
   export ANTHROPIC_API_KEY=sk-ant-...
   claude
   ```

2. **Print Mode fuer sensible Arbeit**:
   ```bash
   claude -p "query"  # No persistent session
   ```

3. **Permission Modes nutzen**:
   ```bash
   claude --permission-mode plan  # Ueberpruefen vor Execution
   ```
